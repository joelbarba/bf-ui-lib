Once the radio is selected, it's bfValue will be set to the ngModel. The selection (using the same ngModel) is unique, and when a radio is selected the rest are automatically unselected.
It is possible to have different radio groups using the same ngModel with multiple radios selected. To do so, use the bfRadioGroup option, assigning different names for every group.

By default the radios are not required. This means that if no option is selected (model matches no value) the form that wraps all the inputs will still be valid. If required, the form will only be valid if the model matches one radio.
For more details about the original bootstrap radio: https://getbootstrap.com/docs/4.0/components/forms/#checkboxes-and-radios
