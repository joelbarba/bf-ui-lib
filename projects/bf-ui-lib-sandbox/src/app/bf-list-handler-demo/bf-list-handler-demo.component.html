This class generates an object that encapsulates the default behaviour we usually need to render a list in the UI.

The object generated with ListHandler is a “mini reducer” that handles the state for the list and all its configuration. It provides a final observable render$ which we can subscribed to and react to any change on the rendered list.
The render$ returns the whole configuration object. If we want to subscribe only to the list we can use renderList$, which returns the list to render on the page (renderedList) so we can easily use | async.

There are 4 different types of events that can trigger a change on a rendering of the list:

  • content → Should the source of the list changes (data).
  • filter → If the user filter results of the list throuhg the UI.
  • order → If the user changes the order of the list
  • paginate → In case we paginate the list and want to change any of the pagination properties (items per page / current number)
