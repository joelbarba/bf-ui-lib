Asynchronous Handling

In order to handle asynchronous tasks triggered by buttons, there's an option to automatically block all buttons until an async process is resolved.
That is helpful to avoid unwanted cases like calling an API twice, etc.

To do so, just pass the promise that resolves the async process to [bfAsyncPromise]. 

{{ asyncExample1 }}

It is also possible to do the same using [bfAsyncClick]. Passing a callback function to this parameter, you don't need to use (bfClick) output.
That callback should return the promise that blocks the button till resolving. That function is called into the component, so to pass parameters you should bind them on its definition:

{{ asyncExample2 }}

CSS Styling

Every name in "$bf-ui-lib-colors" represents a css class that you can apply on the button to color it (primary, secondary, tertiary, quaternary, warning, extra, white, disabled)

To change the color of the buttons use the SASS variable $default-btn-text-color

To add extra colors, there is a mixin that can be used inside a selector "custom-btn-color(color-bg, text-color)":
